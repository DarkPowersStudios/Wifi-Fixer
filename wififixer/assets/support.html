Support:

As an Open Source project, Wifi Fixer depends on you to help make it better. 
Constructive feedback, ideas, and problem reports are always appreciated.  When emailing the developer, please try to be specific and descriptive.

Wifi Fixer has a logging feature which lets you send diagnostic logs directly to the developer with one tap.  The log's contents are helpful in diagnosing issues you might have.
Logging controls are available in Wifi Fixer's preferences as one of the categories.
The log file is stored in your sdcard under /Android/data/org.wahtod.wififixer/files/wififixer_log.txt   Some older devices may put the file in a different location.

Note: Data is only captured to the log when you enable logging.   The log's data is a recording of what wifi and Wifi FixerService are doing, so you must leave it on to record the intended issue and turn it off after the issue is recorded, much like a video camera.   The service must be turned on to capture data to the log.

Since version 0.9.0, Wifi Fixer has recorded stack traces from crashes. 
If you get a force close,  tap Send Log to send the stack trace to the developer.
Stack traces contain no personal information of any kind.  

NOTES:
Please leave logging on for a few minutes.   What happens before and after what you are trying to capture may be useful in diagnosing the issue.
When you send the log, issue reports are important.  Please include specific time  the issue occurred so the developer can find it in the log.
